3. The Three KP Regression Datasets

3.1 The Development Index Dataset

kp_subdistrict_development_index.csv

Concept: Suppose someone collected survey data from 100,000 communities in KP (terrain type, literacy rate, school enrollment, income, access to electricity). Now he wants to find the development_score (0 to 100). What do we do? We fit linear regressions, checking for high multicollinearity.

  • Target: development_score
  • Identifiers: community_id (Should be dropped!)
  • Key Challenge: Severe collinearity between literacy and school enrollment.

3.2 The Agricultural Yields Dataset

kp_agricultural_yields.csv

Concept: Suppose a researcher collected farm-level metrics (pH, fertilizer use, organic flags, rainfall, temperature, crop types). Now he wants to find crop_yield_tons_per_acre. What do we do? We engineer non-linear curves because soil pH has an optimal range and temperature effects depend on the crop type.

  • Target: crop_yield_tons_per_acre
  • Identifiers: farm_id
  • Key Challenge: Complex interactions and non-linear response profiles.

3.3 The Infrastructure Projects Dataset

kp_infrastructure_projects.csv

Concept: Suppose the KP government logged 100,000 historical public works. Now they want to find final actual_cost_million_pkr and actual_duration_months. What do we do? We build multi-output models and apply log-transforms to counter severe right-skew.

  • Targets: actual_cost_million_pkr, actual_duration_months
  • Identifiers: project_id
  • Key Challenge: Multi-output targets with highly skewed cost distributions.

Chapter 0 Exercises

Exercise 0.1: The Data Handshake

Write a Python script verify_setup.py to load all three datasets and print their shapes, columns, and target variable ranges (minimum and maximum values).

Helpful Comment: Use pd.read_csv() and df.describe(). If it fails with FileNotFoundError, make sure you ran the generation script. Data does not create itself out of magic!

Exercise 0.2: Spotting the Imposters (Identifiers)

Review all columns in the three datasets. Why must we drop ID columns like community_id before feeding data to a linear model? What happens if the model memorizes these IDs?

Helpful Comment: Memorizing unique row IDs results in perfect training scores but absolute failure on test splits. Drop them immediately!

Exercise 0.3: Tracing the Data-Generating Code

Open generate_data.py and inspect the generate_development_dataset function. Write down the exact weights assigned to literacy_rate and public_funding_allocated_million_pkr when computing raw_score.
