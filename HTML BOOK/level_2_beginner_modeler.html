Day 11: First Regression Models — Building the Baselines

Suppose someone collected data from communities in KP ($X$, containing attributes like population, utilities access, etc.) and they want to predict the development_score ($y$). We start with three baseline choices:

DUMMY REGRESSOR

Dummy baseline

Always predicts the training mean target. If your ML model can't beat this, something is wrong.

LINEAR REGRESSION

The classic line

Fits an additive linear model: $\hat{y} = \beta_0 + \beta_1 x_1 + \dots$. Read about the mathematical Ordinary Least Squares Derivation.

DECISION TREE

First curves

Splits data into boxes (e.g. population > 5000) and predicts the mean in each box.

Day 11 Recap

  • Always build a baseline first to establish a performance floor.
  • DummyRegressor predicts a constant target mean or median.
  • LinearRegression models linear, additive relationships.
  • Shallow DecisionTreeRegressor partitions features into region averages.

Day 12: Model Training, Validation, and Testing

How do we evaluate if our model is ready for deployment? Testing on training data is cheating. We split our data into roles:

TRAIN SET (60%)

Parameter fitting

Used to fit parameters (e.g., learn OLS coefficients or decision tree splits).

VALIDATION SET (20%)

Hyperparameter tuning

Used to compare models, select features, and tune hyperparameters (e.g., max_depth).

TEST SET (20%)

Honest evaluation

Locked in a vault. Used exactly once at the very end to evaluate true generalization.

12.1 The Danger of Test Set Leakage

Information leaks when test set patterns influence model parameters, hyperparameter tuning, or preprocessing (e.g., fitting StandardScaler on the entire dataset). Keep your test set locked away.

12.2 Group Splits (Spatial Generalization)

If villages in the same district share unobserved similarities, random splits leak district biases. To check how a model generalizes to a new district, we must hold out entire districts using GroupKFold.

12.3 Time Splits (Temporal Generalization)

For infrastructure projects or other time-series data, random splitting leaks the future. We must train on past years (e.g. 2015-2022) and test on future years (e.g. 2023-2025).

Day 12 Recap

  • Separate data into Train (fit weights), Validation (tune settings), and Test (final score).
  • Test leakage occurs when test information sneaks into training pipelines or tuning loops.
  • Group splits test spatial generalization to unseen geographic regions.
  • Temporal splits test chronological forecasting and prevent future-to-past leakage.

Day 13: Evaluation Metrics — How to Judge a Model

How do we quantify error? Different metrics punish errors in different ways:

Mean Absolute Error (MAE)

mean(|y - ŷ|)

Average absolute prediction error. Easy to explain, scale-linear, robust to outliers.

Root Mean Squared Error (RMSE)

sqrt(mean((y - ŷ)²))

Squares errors before averaging. Punishes larger errors heavily. Ideal when big errors are catastrophic.

R-squared ($R^2$)

1 - (RSS/TSS)

The proportion of variance explained. 1.0 is perfect, 0.0 is dummy mean prediction, and negative is worse than dummy.

Day 13 Recap

  • MAE measures average absolute error scaling linearly.
  • RMSE squares errors, making it highly sensitive to outliers.
  • R-squared measures performance relative to predicting the training mean.

Chapter 2 Recap: Setting the Standards

  • Baseline is the Floor: Always fit a Dummy Regressor first to establish a minimum performance floor.
  • Splitting prevents memorization: Train/Val/Test splits, Group splits, and Temporal splits check spatial and temporal generalization.
  • Leakage is cheating: Ensure preprocessing parameters are fit solely on the training split to keep tests honest.
  • Metric Selection: Choose MAE for robust linear scaling; choose RMSE to quadratically punish outliers.

Formula Cheat Sheet

Concept Formula / Rule Day
Dummy Prediction (Mean) $\hat{y}_i = \bar{y} = \frac{1}{n}\sum y_j$ 11
Linear Prediction $\hat{y}_i = x_i^T\beta = \beta_0 + \beta_1 x_{i1} + \dots$ 11
Mean Absolute Error (MAE) $\text{MAE} = \frac{1}{n}\sum_{i=1}^n \vert y_i - \hat{y}_i\vert$ 13
Root Mean Squared Error (RMSE) $\text{RMSE} = \sqrt{\frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)^2}$ 13
R-squared ($R^2$) $R^2 = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$ 13
